# دليل استخدام واختبار تطبيق نور الإسلام

<p align="right"><strong>للمستخدمين ومسؤولي الفروع</strong><br>
<strong>نظام Windows — واجهة عربية (من اليمين إلى اليسار)</strong></p>

---

## مقدمة

هذا الدليل يساعدك على **تجربة التطبيق خطوة بخطوة** والتأكد أن كل شيء يعمل كما تتوقع: تسجيل الدخول، المبيعات، المخزون، التحويلات، المرتجعات، والتقارير.

اقرأ الجداول من **اليمين إلى اليسار**: عمود **#** ثم **ماذا تفعل؟** ثم **ماذا يجب أن ترى؟** ثم ضع علامة في **تم؟** (✓ نجاح، ✗ مع ملاحظة).

**اسم التطبيق على الشاشة:** نور الإسلام

---

## قبل أن تبدأ

| ما تحتاجه | ملاحظة |
|:----------|:-------|
| جهاز Windows | التطبيق مخصص لسطح المكتب |
| اتصال إنترنت | مطلوب لمعظم العمل (ما عدا البيع المؤقت دون نت) |
| اسم المستخدم وكلمة المرور | من مسؤول النظام |
| طابعة إيصالات (اختياري) | لطباعة فواتير البيع |

|:-:|:-----------|:------------------|:---:|

| 4 | المحتوى في الوسط | داخل إطار أبيض مرتب، وليس ملاصقاً للحواف | |

## 2. تسجيل الدخول والخروج

| # | ماذا تفعل؟ | ماذا يجب أن ترى؟ | تم؟ |
|:-:|:-----------|:------------------|:---:|
| 1 | أدخل البريد وكلمة المرور الصحيحة واضغط دخول | تفتح **لوحة التحكم**؛ شعار التطبيق على شاشة الدخول | |

## 3. لوحة التحكم (الصفحة الرئيسية)

هذه الصفحة تعطيك **نظرة سريعة** على وضع المحل.

| # | ماذا تفعل؟ | ماذا يجب أن ترى؟ | تم؟ |
|:-:|:-----------|:------------------|:---:|
| 1 | افتح **لوحة التحكم** من القائمة | تحميل البيانات بدون رسالة خطأ حمراء | |
| 2 | اقرأ بطاقة **ربح الأسبوع** | رقم الربح ونسبة الهامش، مع إيراد وتكلفة وربح الأسبوع | |
| 3 | اقرأ **مبيعات القطع (رسم بياني)** | خطوط لأكثر القطع مبيعاً **شهراً بشهر** خلال السنة؛ أسفلها أسماء القطع بألوان | |
| 4 | اضغط **التقرير الكامل** تحت الرسم | تفتح تقرير المبيعات التفصيلي للقطع | |
| 5 | اقرأ تنبيهات الأعلى (إن وُجدت) | مثلاً: مخزون منخفض، أقساط متأخرة | |
| 6 | اضغط **فتح نقطة البيع** | تنتقل لشاشة البيع | |
| 7 | اضغط أيقونة التحديث | الأرقام تتحدث من الخادم | |

**لفهم الأرقام:**

- **ربح الأسبوع** = ما كسبته هذا الأسبوع تقريباً.
- **إيراد الأسبوع** = مجموع مبيعات الأسبوع.
- **تكلفة الأسبوع** = تكلفة البضاعة المباعة.
- **الرسم البياني** = أي قطعة باعت أكثر وفي أي شهر.

## 4. القطع (الأصناف / البضاعة)

هنا تُعرَّف المنتجات التي تبيعها (كود، اسم، سعر بيع، سعر تكلفة).

| # | ماذا تفعل؟ | ماذا يجب أن ترى؟ | تم؟ |
|:-:|:-----------|:------------------|:---:|
| 1 | افتح **القطع** | قائمة: كود — اسم — سعر | |
| 2 | ابحث بكود أو اسم | تظهر النتائج المطابقة فقط | |
| 3 | اضغط على قطعة (وليس أيقونة التعديل فقط) | تفتح **تحليل القطعة** (مخزون، مبيعات، رسم شهري) — يحتاج نت | |
| 4 | **قطعة جديدة** | نافذة: كود، اسم، **فئة** و**وحدة** من قوائم (من الخادم)، أسعار، **صورة اختيارية** | |
| 5 | احفظ قطعة جديدة | تظهر في القائمة؛ صورة مصغرة إن رُفعت | |
| 6 | أيقونة **تعديل** | تغيير الاسم أو الأسعار أو الفئة أو الصورة | |
| 7 | صورة أكبر من 2 ميجا أو غير JPG/PNG/WebP | رسالة رفض قبل الرفع | |

## 6. التحويلات بين الفروع

نقل بضاعة من فرع إلى فرع آخر.

| # | ماذا تفعل؟ | ماذا يجب أن ترى؟ | تم؟ |
|:-:|:-----------|:------------------|:---:|
| 1 | **تحويل جديد** | | |
| 1.1 | **من فرع** و **إلى فرع** من القوائم | أسماء فروع | |
| 1.2 | **القطعة** من القائمة | كود واسم | |
| 1.3 | **الكمية** ثم **إنشاء** | تحويل «معلق» | |
| 1.4 | نفس الفرع في «من» و«إلى» | رسالة: فرعان مختلفان | |
| 2 | في القائمة | من [فرع] → إلى [فرع] | |
| 3 | إكمال ✓ لتحويل معلّق | نافذة: تقييم (تكلفة/بيع) + تسجيل مستحق بين الفروع | |
| 4 | أكمل مع تفعيل المستحق | رسالة نجاح؛ يظهر في **حسابات الفروع** | |

| التقرير | ماذا يفيدك؟ | ماذا تفعل؟ |
|:--------|:------------|:-----------|

| **أرصدة العملاء** | من عليهم ديون | **عرض التقرير** |
| **ديون الموردين** | ما تدينون | **عرض التقرير** |

| **مبيعات القطع (رسم بياني)** | أفضل القطع شهرياً | سنة + ترتيب + **عرض التقرير** |

| # | الخطوة | تم؟ |
